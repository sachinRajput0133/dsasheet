🗺️ Route 53  DNS
Like a phone book. Translates "dsasheet.com" into the real IP address of the server. Runs in Amazon's backbone — extremely fast and reliable.
routes to
⚡ CloudFront  CDN
A network of servers around the globe. If you're in Mumbai, your images load from a Mumbai copy — not a server in the US. Handles 99% of file requests without touching the main server.
🪣 S3 Bucket  Storage
Amazon's file storage. All the HTML, CSS, JS, images live here. CloudFront reads from S3 and caches them near users.
API requests only (/api/*)
🖥️ EC2 Instance — your main server (t3.medium)
🚦 Nginx  Gatekeeper
The front door of the server. Does 3 jobs: (1) handles HTTPS/SSL so the connection is encrypted, (2) blocks attackers who send too many requests (rate limiting), (3) forwards the request to the right app.
⚙️ NestJS API  Backend :3001
The brain. Handles all /api/* requests: login, progress saves, fetching problems. Speaks to the database. Runs in a Docker container.
🖼️ Next.js  Frontend :3000
Renders the pages for the browser. Can do Server-Side Rendering (builds the page on the server) or Static pages. Also in Docker.
secure connection (VPC peering)
🍃 MongoDB Atlas  Database
The memory of the app. Stores users, problems, your progress. Runs as a 3-node Replica Set — meaning 3 copies of the data exist. If one server dies, another takes over instantly. No data loss.
push to main → auto-deploy
🐙 GitHub Actions  CI/CD
When a developer pushes code, this runs automatically: tests it, builds it, and deploys it to AWS. Zero manual steps.
📦 ECR  Docker Registry
Amazon's private Docker image store. GitHub Actions builds a container image and pushes it here. EC2 pulls the latest image on deploy.

🚀 How This Handles 1,000,000 Users

⚡

CloudFront takes the hit

Static files (HTML, CSS, JS) are served from 200+ edge locations worldwide. 99% of traffic never reaches your EC2 server at all.

🧱

Stateless API = horizontal scale

The NestJS server holds zero memory of users between requests. Need more capacity? Just clone the EC2 and put a Load Balancer in front. No code changes needed.

🔑

JWT — no session storage

Your login token lives in your browser. The server doesn't store who is logged in. This means any server in the cluster can answer any request.

📇

Database indexes

MongoDB has indexes on all key fields (email, topicId, userId+problemId). Finding a record in a 10M-row table takes microseconds, not seconds.

⚛️

Optimistic UI

When you tick a checkbox, the UI updates instantly before the server responds. Even on a slow connection it feels instant.

🚦

Rate limiting at the gate

Nginx blocks any IP that sends more than 20 login requests/minute. Bots and attackers can't flood the server.

♻️

MongoDB Replica Set

3 database copies. Reads can be served from any replica — spreading load. If the primary fails, a new primary is elected in seconds.

🗂️

CDN Cache headers

Static files are cached for 1 year (max-age=31536000, immutable). Once a user downloads them, their browser never re-downloads them until the filename changes.

🚶 Journey of a Single Request: "Mark problem as solved"

1

You click the checkbox

UI updates immediately (optimistic update). You don't wait for the server — it feels instant.

2

Browser sends POST /api/v1/progress/toggle to CloudFront

The request travels to the nearest CloudFront edge. For API calls, CloudFront forwards to EC2 (it doesn't cache API responses).

3

Nginx receives the request

Checks: Is this IP rate-limited? Is SSL valid? Then forwards to NestJS on port 3001.

4

NestJS validates your JWT token

Reads your token, checks it's not expired, identifies your userId. No database call needed — the token is self-contained.

5

MongoDB upsert in one write

Looks up {userId, problemId} via compound index (super fast). Writes "completed = true" in a single atomic operation. No duplicates possible.

6

Response travels back

NestJS → Nginx → CloudFront → Your browser. The UI was already updated at step 1, so you see nothing change. If the server failed, the checkbox would revert.

🔄 Deployment Pipeline — What Happens When You Push Code

1

Developer pushes to main branch on GitHub

GitHub Actions workflow triggers automatically.

2

CI: lint + build check

Every PR runs lint and build. Broken code cannot merge to main.

3

Frontend: Next.js build → S3 sync

Builds the static site, uploads all files to S3 bucket. Then invalidates CloudFront cache so users get the new version immediately.

4

Backend: Docker image → ECR

Builds a Docker container of the NestJS API, pushes it to Amazon's container registry (ECR).

5

EC2: SSH in, pull new image, restart

GitHub Actions SSHs into the EC2 server, pulls the new Docker image from ECR, and restarts the container. Zero code touches the server manually.
